What it is

Superpowers is an agentic skills framework and a software-development methodology. Its own description calls it "a complete software development methodology for your coding agents, built on top of a set of composable skills and some initial instructions." Built by Jesse Vincent (obra), it installs into your coding agent and teaches it to work like a careful senior engineer before it writes any code.

The core idea is that the agent — not a Python orchestrator or a hidden runtime — is the intelligence. Superpowers ships mostly as Markdown: each "skill" is an instruction file the agent reads on demand, plus a small amount of shell and JavaScript to register those skills with each host tool. There is no server to run and no model to host; the behavior change comes entirely from the instructions the agent loads.

How it works

A gateway skill, using-superpowers, acts as a decision gate: before responding or acting, the agent is told to check whether any skill applies and to invoke it first. Process skills (brainstorming, systematic debugging) set the approach; implementation skills then carry it out. A typical feature runs through a pipeline of skills:

brainstorming            (clarify the design with the human)
  → using-git-worktrees  (isolate the work)
  → writing-plans        (write an explicit, approved plan)
  → executing-plans / subagent-driven-development
      (TDD: RED → GREEN → REFACTOR, often via subagents)
  → requesting-code-review / receiving-code-review
  → verification-before-completion
  → finishing-a-development-branch

The methodology leans on test-driven development, systematic (not ad-hoc) approaches, complexity reduction, and evidence-based verification. Once a plan is approved, the agent is expected to follow it without drifting — the project describes agents working for a couple of hours at a stretch while staying on plan.

Skills are just Markdown

Every capability is a readable SKILL.md you can inspect before trusting it — no compiled black box.

Composable by design

Process skills and implementation skills stack, so one request can chain brainstorming, planning, TDD, and review.

Test-driven by default

The TDD skill enforces the RED-GREEN-REFACTOR loop rather than leaving testing to the agent's discretion.

Subagent-driven

Work can be dispatched to parallel subagents in isolated git worktrees, with code review folded into the loop.

Multi-harness

The same skills install into Claude Code, Cursor, Codex, Copilot CLI, Antigravity, and more.

Extensible

A writing-skills meta-skill teaches the agent how to author new skills in the same format.

A separate community repository, obra/superpowers-skills, and an experimental obra/superpowers-lab extend the set with additional, editable skills.

Installing it

Superpowers is distributed as a plugin, so each agent harness installs it separately. A few of the officially documented commands:

HarnessInstall
Claude Code (official)/plugin install superpowers@claude-plugins-official
Claude Code (project marketplace)/plugin marketplace add obra/superpowers-marketplace → /plugin install superpowers@superpowers-marketplace
Cursor/add-plugin superpowers
Antigravityagy plugin install https://github.com/obra/superpowers
GitHub Copilot CLIcopilot plugin marketplace add obra/superpowers-marketplace → copilot plugin install superpowers@superpowers-marketplace
Kimi Code/plugins install https://github.com/obra/superpowers
Pipi install git:github.com/obra/superpowers

Codex, Factory Droid, and OpenCode are also supported with their own procedures — check the repo's README for the current, exact commands, since the marketplace names change as the project evolves.

Field notes from a hands-on inspection

Because Superpowers works by injecting instructions your coding agent then follows, it's worth a look before wiring it into a tool with file and shell access. A read of the public main branch found:

  • ✓Transparent, readable skills. The capabilities are plain Markdown SKILL.md files, not compiled artifacts. You can read exactly what each skill instructs the agent to do before enabling it.
  • ✓Permissive, well-known license. The project is MIT licensed, so there are no unusual redistribution obligations.
  • ✓Active and widely used. It's one of the more popular agent-tooling projects on GitHub, with a public issue tracker, a community skills repo, and frequent updates.
  • ✓Sensible defaults. The methodology's own guardrails — mandatory planning, TDD, explicit human approval during brainstorming — push toward safer, reviewable work rather than unchecked automation.
Honest caveats for an instruction-injection framework: the using-superpowers skill is deliberately forceful ("if there's even a 1% chance a skill applies, use it"), so the plugin meaningfully steers your agent — read the skills you're enabling. It installs hooks and registers with each harness, and the project collects optional version-adoption telemetry you may want to review or disable. As with any agent plugin, skills can change between releases, so re-skim them after updates and run the agent with the least access it needs.
